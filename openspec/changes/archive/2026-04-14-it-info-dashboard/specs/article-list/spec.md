## ADDED Requirements

### Requirement: 取得済み記事を一覧表示できる
システムは取得済みの記事を一覧で表示しなければならない（SHALL）。各記事にはソース・タイトル・公開日・タグ・既読状態・お気に入り状態を表示する。

#### Scenario: 初期表示
- **WHEN** ユーザーがダッシュボードのトップページにアクセスする
- **THEN** DB に保存されている全記事が公開日の降順で表示される

#### Scenario: 記事が0件のとき
- **WHEN** DB に記事が1件も存在しない状態でページにアクセスする
- **THEN** 「記事がありません。スクレイプボタンで取得してください」旨のメッセージが表示される

### Requirement: ソースでフィルタリングできる
システムは指定されたソース種別の記事のみを表示できなければならない（SHALL）。フィルター状態は URL search params（`?source=`）で管理する。

#### Scenario: ソースフィルター適用
- **WHEN** ユーザーがソースフィルター（例: Zenn）を選択する
- **THEN** 選択したソースの記事のみが表示され、URL に `?source=zenn` が反映される

#### Scenario: フィルターなし（全件表示）
- **WHEN** ソースフィルターが選択されていない
- **THEN** 全ソースの記事が表示される

### Requirement: ステータスでフィルタリングできる
システムは既読・未読・お気に入りのステータスで記事を絞り込めなければならない（SHALL）。

#### Scenario: 未読フィルター
- **WHEN** ユーザーが「未読」フィルターを選択する
- **THEN** `isRead = false` の記事のみが表示される

#### Scenario: お気に入りフィルター
- **WHEN** ユーザーが「お気に入り」フィルターを選択する
- **THEN** `isFavorite = true` の記事のみが表示される
