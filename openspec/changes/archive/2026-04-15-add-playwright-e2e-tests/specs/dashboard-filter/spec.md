## ADDED Requirements

### Requirement: ソースフィルターがURLクエリパラメータを更新する
ソースフィルターボタンをクリックすると、システムは URL の `source` クエリパラメータを更新し、選択されたフィルターボタンをアクティブ状態（青背景）で表示しなければならない（SHALL）。

#### Scenario: ソースフィルターをクリックするとURLが更新される
- **WHEN** ユーザーが「Zenn」ソースフィルターボタンをクリックする
- **THEN** URLに `?source=ZENN` が含まれる

#### Scenario: アクティブなソースフィルターのスタイルが変わる
- **WHEN** ユーザーが「Qiita」ソースフィルターボタンをクリックする
- **THEN** 「Qiita」ボタンが青背景（アクティブ状態）になり、他のボタンは非アクティブ状態になる

#### Scenario: 「すべて」をクリックするとソースフィルターが解除される
- **WHEN** ソースフィルターが選択された状態でユーザーが「すべて」ボタンをクリックする
- **THEN** URLから `source` パラメータが削除される

### Requirement: 状態フィルターがURLクエリパラメータを更新する
状態フィルターボタンをクリックすると、システムは URL の `status` クエリパラメータを更新し、選択されたフィルターボタンをアクティブ状態で表示しなければならない（SHALL）。

#### Scenario: 状態フィルター「未読」をクリックするとURLが更新される
- **WHEN** ユーザーが「未読」状態フィルターボタンをクリックする
- **THEN** URLに `?status=unread` が含まれる

#### Scenario: 状態フィルター「既読」をクリックするとURLが更新される
- **WHEN** ユーザーが「既読」状態フィルターボタンをクリックする
- **THEN** URLに `?status=read` が含まれる

#### Scenario: 状態フィルター「お気に入り」をクリックするとURLが更新される
- **WHEN** ユーザーが「お気に入り」状態フィルターボタンをクリックする
- **THEN** URLに `?status=favorite` が含まれる

#### Scenario: ソースと状態フィルターを同時に選択できる
- **WHEN** ユーザーが「Zenn」ソースと「未読」状態を順にクリックする
- **THEN** URLに `source=ZENN` と `status=unread` の両パラメータが含まれる
