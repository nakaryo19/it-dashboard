## ADDED Requirements

### Requirement: 記事をお気に入り登録・解除できる
システムは記事ごとにお気に入り（isFavorite）フラグを管理しなければならない（SHALL）。ユーザーがお気に入りボタンを押すと状態がトグルされ、DB に永続化される。

#### Scenario: 記事をお気に入りに追加する
- **WHEN** ユーザーが未登録記事のお気に入りボタンをクリックする
- **THEN** 該当記事の `isFavorite` が `true` に更新され、UI にお気に入り状態（例: 星アイコン塗りつぶし）が即時反映される

#### Scenario: お気に入りを解除する
- **WHEN** ユーザーがお気に入り登録済み記事のお気に入りボタンをクリックする
- **THEN** 該当記事の `isFavorite` が `false` に更新され、UI に未登録状態が即時反映される

#### Scenario: お気に入り状態の永続化
- **WHEN** ユーザーがお気に入り状態を変更した後、ページをリロードする
- **THEN** 変更後のお気に入り状態が維持されている

### Requirement: お気に入り記事を一覧で確認できる
システムはお気に入りフィルターを適用してお気に入り記事のみを表示できなければならない（SHALL）。

#### Scenario: お気に入り一覧の表示
- **WHEN** ユーザーが「お気に入り」フィルターを選択する
- **THEN** `isFavorite = true` の記事のみが一覧表示される

#### Scenario: お気に入りが0件のとき
- **WHEN** お気に入りフィルターが選択されているが、お気に入り記事が存在しない
- **THEN** 「お気に入りに登録した記事はありません」旨のメッセージが表示される
