## ADDED Requirements

### Requirement: 記事の既読・未読状態を管理できる
システムは記事ごとに既読（isRead）フラグを管理しなければならない（SHALL）。ユーザーが記事をクリックまたは既読ボタンを押すと状態がトグルされ、DB に永続化される。

#### Scenario: 未読記事を既読にする
- **WHEN** ユーザーが未読記事の既読ボタンをクリックする
- **THEN** 該当記事の `isRead` が `true` に更新され、UI に既読状態が即時反映される

#### Scenario: 既読記事を未読に戻す
- **WHEN** ユーザーが既読記事の既読ボタンをクリックする
- **THEN** 該当記事の `isRead` が `false` に更新され、UI に未読状態が即時反映される

#### Scenario: 状態変更の永続化
- **WHEN** ユーザーが既読状態を変更した後、ページをリロードする
- **THEN** 変更後の既読状態が維持されている

### Requirement: 記事タイトルクリックで自動既読
システムは記事タイトルリンクをクリックしたとき、該当記事を自動的に既読にしなければならない（SHALL）。

#### Scenario: タイトルクリックによる自動既読
- **WHEN** ユーザーが記事タイトルのリンクをクリックする
- **THEN** 記事が新しいタブで開かれ、該当記事が既読状態に更新される
